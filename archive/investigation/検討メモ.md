# 滞留在庫可視化 検討メモ

## 概要

滞留在庫を可視化し、在庫管理の効率化を図るためのダッシュボード開発プロジェクト。

---

## 1. 目的・ゴール

- [ ] 滞留在庫の現状を可視化する
- [ ] 滞留リスクの早期発見
- [ ] 在庫処分の意思決定を支援

---

## 2. 検討事項

### 2.1 滞留在庫の定義

| 項目 | 検討内容 | 決定事項 |
|------|----------|----------|
| 滞留期間の閾値 | 90日？180日？365日？ | 未定 |
| 判定基準 | 最終販売日から？入荷日から？ | 未定 |
| 除外条件 | 季節商品、予約商品などの扱い | 未定 |

### 2.2 データソース

| 項目 | 検討内容 | 決定事項 |
|------|----------|----------|
| 使用DB | BigQuery / その他 | 未定 |
| 参照テーブル | 在庫テーブル、販売履歴など | 未定 |
| 更新頻度 | 日次？週次？ | 未定 |

### 2.3 可視化項目

- [ ] 滞留在庫一覧（商品別）
- [ ] カテゴリ別集計
- [ ] 滞留期間別の分布
- [ ] 金額ベースの影響度
- [ ] 時系列推移

### 2.4 ダッシュボードツール

| 候補 | メリット | デメリット |
|------|----------|------------|
| Looker Studio | 無料、BigQuery連携容易 | カスタマイズ限界 |
| Tableau | 高機能 | ライセンス費用 |
| スプレッドシート | 手軽、共有容易 | 大量データに弱い |
