## このダッシュボードでできること

このダッシュボードは、**減損が予定されている在庫**を一覧で確認できるツールです。

### 主な用途

| 目的 | 確認すべきポイント |
|---|---|
| **減損回避** | 取得原価が高い商品から優先的に対策を検討 |
| **滞留解消** | 在庫数が多い商品、長期滞留している商品を特定 |
| **期末準備** | 次の期末までに減損予定の在庫を事前把握 |

### 減損とは？

在庫が**365日以上**倉庫に滞留すると、会計上「減損」として処理されます。
減損処理は**期末（2月末/5月末/8月末/11月末）**に実施されます。

## 基本的な使い方

### 1. 除外対象を非表示にする

データには分析対象外の商品が含まれています。
まず**「除外フラグ」列でフィルター**をかけて、「除外」を非表示にしてください。

```
手順:
1. 「除外フラグ」列のフィルターアイコンをクリック
2. 「除外」のチェックを外す
3. 「空白」のみにチェックを入れてOK
```

### 2. 必要な列だけ表示する

すべての列を見る必要はありません。**目的に応じて必要な列だけ表示**すると見やすくなります。

### 3. 画像を表示する（スプレッドシートの場合）

「画像リンク」列のURLをそのまま見るのは大変です。
画像を表示するには以下の方法があります：

**方法A: IMAGE関数を使う**
```
=IMAGE(A2)  ※A2に画像リンクが入っている場合
```

**方法B: 画像貼り付け用シートを用意する**
別シートにIMAGE関数を設定しておき、商品IDで参照すると見やすくなります。

## 目的別の活用方法

### 🎯 目的1: 減損回避（金額インパクトを減らしたい）

**見るべき列**: 取得原価（期末1〜4）

**手順**:
1. 「除外フラグ」で「除外」を非表示
2. **「期末1_減損予定_取得原価」を降順（大きい順）で並べ替え**
3. 上位の商品から減損回避策を検討

**ポイント**:
- 取得原価が高い商品 = 減損時の金額インパクトが大きい
- まず「期末1」（次の期末）を確認し、急ぎの対応が必要か判断

---

### 🎯 目的2: 滞留解消（在庫を動かしたい）

**見るべき列**: 商品数、余りパーツ数、累計

**手順**:
1. 「除外フラグ」で「除外」を非表示
2. **「期末1_減損予定_商品+余りパーツ累計」を降順で並べ替え**
3. 在庫数が多い商品から滞留解消策を検討

**ポイント**:
- 「商品数」= 完成品として組み上げられる在庫
- 「余りパーツ数」= 商品として組み上げられない余りの部品
- 「累計」= その期末までに減損予定の合計（期末1の累計 = 期末1のみ、期末2の累計 = 期末1 + 期末2）

---

### 🎯 目的3: 期末ごとの影響を把握したい

**見るべき列**: 期末1〜4の日付、各期末の数量・取得原価

**ポイント**:
- 「期末1_日付」〜「期末4_日付」列で、具体的な期末日を確認できます
- 例: 今日が12月なら、期末1 = 2月末、期末2 = 5月末...

---

### 🎯 目的4: 特定の商品を詳しく調べたい

**見るべき列**: 構成パーツid、構成パーツ名、在庫id

**手順**:
1. 調べたい商品のSKU_IDまたは商品名でフィルター
2. 「構成パーツid」「構成パーツ名」で、何のパーツで構成されているか確認
3. 「在庫id」列で、具体的にどの在庫が対象か確認

**在庫IDの見方**:
```
[パーツid:倉庫名]在庫id, 在庫id, ...

例: [8049:船橋]123456, 123457
    ↑パーツ8049が船橋倉庫にあり、在庫ID 123456と123457が対象
```

## よくある質問

### Q1: 「商品数」と「余りパーツ数」の違いは？

**商品数**: 完成品として組み上げられる在庫の数です。
例えば、ベッドフレームが「本体3点 + 脚1点」で構成される場合、本体6点と脚2点があれば「2商品」組み上げられます。

**余りパーツ数**: 商品として組み上げられない余りのパーツです。
上記の例では、本体が6点で脚が2点の場合、脚が足りないため本体が余ります。

### Q2: 期末1、期末2... はいつのこと？

実行日によって自動的に決まります。
- **期末1**: 次に来る期末日（2月末/5月末/8月末/11月末のうち最も近い日）
- **期末2**: その次の期末日
- **期末3**: 3番目に近い期末日
- **期末4以降**: 4番目以降の期末日

具体的な日付は「期末1_日付」〜「期末4_日付」列で確認できます。

### Q3: 「減損済」とは？

すでに365日以上滞留して減損処理済みの在庫です。
今後の減損対策では「期末1〜4」の数値を見てください。

### Q4: 取得原価の数値は正確？

現在は在庫データの取得原価（cost）を集計しています。
今後、より厳密な**簿価**（会計上の帳簿価額）に更新される予定です。

### Q5: 画像が見えない

「画像リンク」列にはURLが入っています。
スプレッドシートで画像を表示するには、別セルで `=IMAGE(セル参照)` を使うか、
画像表示用の別シートを用意してください。

## 注意事項

### ⚠️ 縦合計は信じないでください

このデータは**SKU単位**で集計されています。
同じ在庫が複数のSKUで使用される場合があるため、**縦方向の合計（SUM）は正確ではありません**。

```
例: パーツAが SKU_ID 10 と SKU_ID 15 の両方に使われている場合、
    両方の行にパーツAの数量・原価が含まれます。
    → 縦合計すると二重カウントになります
```

**正確な合計が必要な場合**は、別途パーツ単位での集計をご依頼ください。

### ⚠️ 「除外フラグ」は必ず非表示に

「除外」となっている行は以下の理由で分析対象外です：
- 削除された商品
- セット商品（個別に計算できないため）
- 特殊な用途の商品
- 重複している商品（同じ構成の商品が複数あり、1つだけ残している）

分析する際は必ず「除外」を非表示にしてください。

### ⚠️ データは実行時点のスナップショット

このデータはクエリを実行した時点の在庫状況です。
日々の入出荷により数値は変動しますので、最新状況を確認したい場合は再実行してください。

- より詳細な分析が必要 → 開発担当者にご相談ください
